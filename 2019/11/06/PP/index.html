类·运算符重载

           当参数主要以类为主的时候，一般会在类的内部定义重载函数(类外实现)。

类内的重载运算符函数：

类内定义

➕ ➖ ✖ ➗ = 的重载

           重载时一般在括号后边加 const 修饰表示类的不变性，参数是类的时候一般在括号内也加const来避免类被修改，最后返回一个在该函数里面定义的临时类变量。但是等于号时括号外边的const不可加，因为我们要改变类的成员函数。

           注意： class=class+具体值 函数的参数类型为该数值的类型。 class=具体值+class 这时候operator函数不再发挥作用，具体值没有重载函数，这种情况下我们在类的外部定义一个函数调换位置来间接引用类内的函数.

流操作重载的说明

           使用流的重载函数时会改变流,因此向函数传递的必须是流的引用，而不是传值。为了提高效率，第二个类的参数的传递也采用传引用的方式，const来保证类的成员函数的值不后悔发生变化。如果不要求多次使用输出流运算符，那么就可以将返回值类型改变为void 最后也不用再返回，因为之后的cout也不必再使用重载后的流运算符。

转换运算符重载

类内定义

           转换运算符与其他函数有两点不同。分别为转换运算符的函数无需实参,转换运算符函数没有返回类型，void也不能用。直接在函数中写operator int()还可以是其他类型。不需要加前缀，就像构造函数一样。

类内的动态内存分配

           在成员函数内定义一个指针类型，一个数值类型。一个用来存数组的地址，另一个用来存储元素个数,； 构造函数定义:

           C++要求我们手动处理内存问题。所以我们采用析构函数来清理我们申请的内存。析构函数——没有形式参数没有返回值。

           析构函数发挥作用的时候是类所在的有效区域结束时。

           使用动态内存分配引起的重载运算符=出现问题。如果只是进行成员的逐个赋值，那么指针的地址也会被复制到另一个类中，这种拷贝类型叫做浅拷贝。这种拷贝方式的麻烦之处在于结束时，当两个类均被销毁时析构函数对指针所指区域delete了两次，这就造成了指令不明确，会发生编译错误。为了避免这种方式到带来的影响，我们使用深拷贝方式，即把内存中的数值拷贝到另 一块连续的数组中去，让指针去指向这个地址。所以，在重载运算符=的函数内我们应该这样书写

           这就完成了深度拷贝；为了提高效率，我们传给函数类的值时采用了传地址的方式。使用const保证不会被修改。我们还有一种方式是采用拷贝构造函数：

拷贝构造函数是一种特殊的构造函数，下列情况发生时都会调用此函数。 • 创建并初始化一个新对象时。 语句：类 参数=类参数（这个可以是一个定义好的已经修改过值的类） • 按值传递的方式将一个对象传递给函数时。 Any_function（类）； 此时触发了拷贝构造函数 • 从函数返回对象本身时 return 类；

           每个类都有一个默认的拷贝构造函数，它与默认的赋值运算符类似逐个成员进行复制。对于大多数类而言，默认的拷贝构造函数和默认的赋值运算符都足以达到了我们的要求。但是，对于需要动态内存分配和释放内存的类，默认的拷贝构造函数与默认的赋值运算符存在同样的问题。通常，一个定义了自己的赋值运算符的类也必须定义它自己的拷贝构造函数。

           这就是拷贝构造函数的实现过程。一个拷贝构造函数只能有一个形参，它是指向同一个类的对象的引用。这个形参是一个const的引用，拷贝构造函数不能通过这个引用来修改传递给它的对象。

错误检测：

           我们访问数组的时候如果超出了数组的范围，那么我们需要让系统来帮我们检测一下错误。重载运算符[]：

           在构造函数中，类使用的指针为数组分配指定大小的空间。重载下表运算符[]只是在返回数组元素前检查下标值。如果下标值无效，显示错误信息并终止程序。exit（0）表示正常运行结束，其他的数值表示非正常结束。

重载运算符规则:

*不能发明新的运算符:
例如用<>来表示不等于或者用表示幂.

*重载运算符后的操作数个数必须与原来的相同:
例如++的操作数是一个，==必须有两个操作数

*运算符重载后还是有原来的优先级

*重载的运算符不能使用默认实参

*用于内置数据类型时(如int，float时)运算符的含义不能被重新定义
